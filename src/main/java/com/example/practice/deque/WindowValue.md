获取滑动窗口最大值和最小值问题。  
有一个正整数数组和一个滑动窗口，窗口从左向右滑动，尽可能快速地求出任意时刻窗口内元素的最大值/最小值。  
窗口可以固定宽度，也可以窗口的左右边界各自独立移动，但左边界一定小于右边界。  

解决方案：双端队列 deque  
1.获取窗口最大值数组  
deque 中存的是部分数组元素的下标，且保证下标对应的值严格从大到小，这样在窗口从左向右移动时，就能始终保持 deque 的 head 存储的是当前窗口的最大值。每次获取到窗口最大值存入最大窗口值数组，最后返回。  

如何做到？  
①窗口右边界 R 向右移动时，新进入窗口的元素 arr[i]，比较 arr[i] 与 deque 尾部下标 last 所对应元素 arr[last]，  
若 arr[i] < arr[last]，arr[i] 从尾部入队；  
若 arr[i] >= arr[last]，将 last 出队，且直到 deque 的新队尾 arr[last]>arr[i]，或者 deque 变为空，此时 i 从队尾入队。  

②窗口左边界 L 向右移动时，刚过期的窗口元素 left-1 （left 是此时窗口左边界元素），比较 arr[left-1] 与 deque.peekFirst()，若相等表示刚过期的元素是窗口最大值，需要 deque.pollFirst()，将队头元素出队，否则，不做处理。  

例子：arr = [1, 3, 9, 7, 6, 4, 0, 5, 23]，固定窗口左边界，右边界每次向右移动一个位置。deque 的变化情况。  
R==0,deque={1}  
R==1,deque={3}  
R==2,deque={9}  
R==3,deque={9,7}  
R==4,deque={9,7,6}  
R==5,deque={9,7,6,4}  
R==6,deque={9,7,6,4,0}  
R==7,deque={9,7,6,4,0,5}  
R==8,deque={23}  
deque 保存数组下标，下标对应的数组元素严格从大到小。  

2.获取窗口最小值数组  
同获取最大值唯一的区别：deque 中下标对应的数组值严格从小到大。deque head 始终保存窗口最大值下标。
